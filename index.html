üìò Module 1: Getting Started with CSS

üéØ Goal:

Understand what CSS is, how it works with HTML, how the browser interprets it, and how to start writing it.

üß† 1.1 What is CSS?

CSS (Cascading Style Sheets) is the language used to describe the presentation (layout, colors, fonts) of HTML documents.

Why CSS matters:

  • Separates content (HTML) from presentation.
  • Makes it easier to maintain, scale, and reuse design logic.
  • Critical for responsive, interactive, and beautiful UIs.

üì¶ Example:

<!-- HTML -->
<p>Hello, CSS!</p>

<!-- CSS -->
p {
  color: blue;
  font-size: 20px;
}

Hello, CSS!

üèó 1.2 How CSS Fits in the Web

The "Trinity": HTML + CSS + JS

HTML

Structure (e.g., "this is a paragraph")

CSS

Presentation (e.g., "make it blue and centered")

JS

Behavior (e.g., "animate it when clicked")

üîå 1.3 Ways to Add CSS

There are 3 main methods:

1. Inline CSS (Avoid for reuse):

<p style="color: red;">Red text</p>

2. Internal CSS (in <style> tag):

<style>
  p { color: red; }
</style>

3. External CSS (Best practice):

<!-- HTML -->
<link rel="stylesheet" href="styles.css" />

/* styles.css */
p {
  color: red;
}

üîç 1.5 What Happens Under the Hood?

  • The browser parses HTML, finds the CSS files.
  • CSS is parsed into a render tree.
  • The browser applies the cascade rules, resolving:
    • Specificity
    • Importance (!important)
    • Order
  • Then it paints the layout on screen.

üí° Best Practices

  • Always use external stylesheets for scalability.
  • Avoid inline styles unless for quick testing or emails.
  • Keep structure (HTML) and design (CSS) separate.

üìò Module 2: CSS Syntax & Selectors

üéØ Goal:

Understand how CSS rules are structured, and how to target any element on a webpage using selectors. Learn specificity and how CSS decides which rule wins when there's a conflict.

üß± 2.1 CSS Rule Syntax

A CSS rule is made of:

selector {
  property: value;
}
  • Selector ‚Äì What HTML element(s) to target
  • Property ‚Äì The style feature you want to change
  • Value ‚Äì The setting for that feature

üß™ Example:

h1 {
  color: green;
  font-size: 32px;
}

üß≤ 2.2 Types of Selectors

üîπ 1. Universal Selector ‚Äì Targets everything

* {
  margin: 0;
  padding: 0;
}

üîπ 2. Type Selector ‚Äì Targets HTML tags

p {
  line-height: 1.6;
}

üîπ 3. Class Selector ‚Äì Targets elements with a class

.card {
  background: #f2f2f2;
  padding: 20px;
}

HTML:

<div class="card">Hello</div>

üîπ 4. ID Selector ‚Äì Targets an element with a specific id

#banner {
  height: 300px;
}

HTML:

<div id="banner"></div>

‚úÖ Best Practice: Use classes for styling. IDs should be unique and used for JavaScript or anchor linking.

üß¨ 2.3 Grouping & Combining Selectors

Group multiple selectors

h1, h2, h3 {
  font-family: 'Segoe UI', sans-serif;
}

Combine selectors (descendant)

article p {
  color: darkgray;
}

This selects all p inside an article.

Child combinator (>)

ul > li {
  list-style: square;
}

Selects only direct li children of ul.

Adjacent sibling (+)

h1 + p {
  margin-top: 0;
}

Selects the first p that follows an h1.

General sibling (~)

h1 ~ p {
  color: gray;
}

Selects all p siblings after h1.

üß™ 2.4 Attribute Selectors

Target elements with specific attributes.

input[type="text"] {
  border: 1px solid #ccc;
}

Other examples:

a[target="_blank"] { color: red; }
img[alt~="logo"] { width: 200px; }

üéö 2.6 Specificity & Cascade (Preview)

Specificity = How CSS decides which rule wins

Selector Type Weight
Inline styles: style="" 1000
ID selectors: #id 100
Class/attributes/pseudo: .class, [attr], :hover 10
Type selectors: div, h1 1
Universal * 0

üß™ Example:

/* Specificity 10 */
.button {
  color: red;
}

/* Specificity 100 */
#submit {
  color: blue;
}

If both apply to the same element, the one with higher specificity wins: #submit ‚Üí blue.

We'll dive deeper into this in Module 13.

üé® 3.1 Working with Colors in CSS

  • Hue: 0‚Äì360¬∞ (a color wheel)

üìè 3.2 CSS Units of Measurement

CSS supports absolute and relative units.

üéØ Absolute Units (Fixed)

Unit Meaning
px Pixels ‚Äî most common unit
cm, mm, in Physical length ‚Äî for print
pt, pc Points and picas ‚Äî typography/print 
font-size: 16px;
margin: 10mm;

‚úÖ Use case: px is best for precision and screen control.

üéØ Relative Units (Flexible)

Unit Relative to
% Parent element
em Font-size of element itself
rem Font-size of root element (html)
vw, vh Viewport width/height
vmin, vmax Smaller/larger of vw/vh
ch Width of the 0 character

üîç Comparison: em vs rem

html { font-size: 16px; }

.container {
  font-size: 1.5rem; /* 24px */
  padding: 2em;      /* 2 √ó container font size = 48px */
}

üß† rem is more predictable for layouts.
üß† em is useful for scalable components (e.g., buttons, modals).

üß† 3.3 Value Types in CSS

In CSS, every property accepts certain value types:

Type Example Used For
Length 16px, 1em font, padding, margin
Color #fff, rgb() color, background
Percentage 80% width, height, transforms
Keywords auto, inherit position, display, font-weight
Functions calc(), url() sizes, gradients, images

üßÆ 3.4 The calc() Function

Combine units and do math directly in CSS:

font-size: calc(1rem + 0.5vw);

‚ö†Ô∏è No commas allowed, and spacing matters.
